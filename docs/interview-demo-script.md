# 面试演示脚本

## 3 分钟版

我这次没有把产品做成一个通用聊天机器人，而是做成「博主 AI 运营助理」。因为 C 端创作者真正愿意付费的点，不是 AI 有多聪明，而是它能不能帮助我更快做出能涨粉、能接商单的内容。

第一步，用户先建立博主档案：平台、赛道、调性、粉丝阶段和品牌 brief。这样 Agent 的输出不会是通用文案，而是围绕账号定位生成。

第二步，用户输入对标 URL，并粘贴标题和正文。首期我不把实时抓取作为强依赖，因为小红书和抖音内容抓取不稳定，MVP 里 URL 负责记录来源，正文负责稳定拆解。

第三步，系统拆解爆款公式，并生成标题、笔记正文、短视频分镜和封面方案。这里体现的是全链路工作台，而不是单点写作工具。

第四步，Agent 给出流量潜力、商单友好度、风险表达和发布时间建议。这个预测端是「助理感」的来源，它会主动告诉用户下一步该改什么。

最后，系统输出结构化日志，可以用于后续复盘和报告导出。首期 MVP 的重点是稳定跑通创作闭环，下一版可以接真实模型 API 和账号历史内容，做更强的个性化。

## 8 分钟版

1. 产品判断：服务 1k-10w 粉上升期博主，价值锚定「多接一单商单」。
2. 用户选择：美妆、母婴、家居、穿搭、个人 IP 专家五类变现密度更高。
3. 工作流演示：博主档案、对标输入、爆款拆解、内容生成、流量评估、数据日志。
4. 稳定性设计：URL 不作为唯一输入，内置样例和规则引擎兜底。
5. 定价：体验版 39 元、创作版 99 元、专业版 199 元，主推 99 元/月 15 篇。
6. 差异化：不是替代剪映、Canva、豆包，而是做它们前面的创作运营层。
