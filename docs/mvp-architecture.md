# C 端 Agent MVP 架构

## 交付目标

网页 Demo 优先，展示完整产品操作平台闭环。首版不强依赖真实 URL 抓取和外部模型，保证面试现场稳定跑通；API 接入作为增强层保留。

## 模块

- 博主档案：平台、赛道、调性、粉丝阶段、品牌 brief。
- 输入端：URL 记录来源，正文/口播稿负责稳定拆解，内置样例保证演示可控。
- 生产端：标题、正文、短视频分镜、封面方向、多平台适配。
- 预测端：流量潜力、商单友好度、合规风险、发布时间和修改建议。
- 数据闭环：结构化日志和演示报告。

## 技术选择

- React + TypeScript + Vite
- 规则引擎作为稳定兜底
- 可插拔模型层，后续可接入大模型 API
- 静态部署，优先 Vercel / Netlify / GitHub Pages

## 发布策略

1. 先用内置样例完成稳定演示。
2. 再接入 API Key，把规则引擎输出升级为真实模型生成。
3. 保留稳定模式，面试时遇到接口异常仍能跑完全流程。
